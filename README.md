# C++ Console IDLE

Decided to do little project using requirements generated by AI to implemnet some core OOP, Abstraction; Design Pattens and Clean Code  principles  

## AI generated task

Task: Design a highly modular, console-based Idle Industrial Simulator that serves as a showcase for Clean Code, SOLID principles, and advanced OOP.  

Core Technical Requirements:
 * SOLID Implementation:
  * Single Responsibility: Separate the Game State, the Production Logic, and the Input Handling.

  * Open/Closed: Designing the system so new "Building Types" or "Global Buffs" can be added without modifying the core Engine.

  * Dependency Inversion: High-level game logic should depend on abstractions (Interfaces), not concrete implementations.

 * Clean Code & Memory: Use modern language-specific best practices (e.g., Smart Pointers in C++, Interfaces in C#/Java) and avoid "God Classes."
